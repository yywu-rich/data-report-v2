1 创建 GitHub Personal Access Token（仅一次）

Token 用于授权脚本写入 data.json。仅 1 个仓库的写权限，安全可控。

展开：详细步骤（推荐 Fine-grained Token，更安全）
  1. 访问 GitHub → Settings → Developer settings → Fine-grained tokens → Generate new token
  2. Token name：随便填，如 data-report-v2
  3. Expiration：选 Custom，填一年后；或选 No expiration（不会过期）
  4. Repository access：选 Only select repositories，勾选 yywu-rich/data-report-v2
  5. Repository permissions：找到 Contents，改成 Read and write（其他都不用动）
  6. 点页面底部 Generate token
  7. 立刻复制 Token（以 github_pat_ 开头），关闭页面后再也看不到
展开：如果 Fine-grained 太麻烦，可以用 Classic Token
  1. 访问 GitHub → Settings → Developer settings → Tokens (classic) → Generate new token (classic)
  2. Note：填 data-report-v2
  3. Expiration：建议选 1 年
  4. scopes：仅勾选 public_repo（如果仓库是 Public）或 repo（如果仓库是 Private）
  5. 点 Generate token，立即复制（以 ghp_ 开头）

2 把 Token 粘贴到这里

Token 仅保存在你这台电脑的浏览器本地（localStorage），不会发到任何服务器。 Token 也会被嵌入到下方书签 URL 中。

3 把书签拖到浏览器书签栏

显示书签栏：Ctrl/Cmd + Shift + B。把下面按钮**拖**到书签栏。

⚡ 一键更新报表（本周）

每周五早上点这个书签即可自动抓"上周五-本周四"的数据。
如果以后采集逻辑升级，不需要重新拖书签，会自动加载最新代码。
但如果换了 Token，需要重新拖一次。

📜 补抓历史数据（一次性）

想把过去几周的数据一次性抓回来？设定下面的周数，然后拖动或直接点按钮。

📜 补抓历史

提示：每周需 8 次 Jira 查询（4 指标 × 本期/同期），4 周大约 30 秒，10 周大约 1 分多钟。
本书签是一次性用的，用完不必拖到书签栏；想再次补抓改下数字直接点这个按钮即可。
已存在的周数据会被合并（保留旧字段，覆盖新抓的字段）。

4 每周怎么用

  1. 登录 jira.mailtech.cn（任意页面都行，必须是登录状态）
  2. 点击书签栏的 ⚡ 一键更新报表
  3. 右上角弹出面板，等几秒看到 "✅ 已成功写入 GitHub"
  4. 30 秒后访问报表页查看新数据：报表首页
